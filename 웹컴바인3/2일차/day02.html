var / let / const

----------------------------------------------------------------------------------

var는 중복선언가능

호이스팅 : 인터프리터가 변수와 함수의 메모리 공간을 선언전에 미리 할당하는 것 - var로 선언한 변수의 경우 호이스팅 시 undefind로 변수를 초기화 합니다.
let과 const로 선언한 변수의 경우 호이스팅 시 변수를 초기화 하지 않습니다.

var 는 function Scoped이고 let, const는 Block Scoped 입니다.
undefind? : 변수는 존재하지만, 어떠한 값으로도 할당 되지 않은 상태(자료형이 정해지지 않은 상태)
null? : 변수는 존재 하나, 값이 null 할당한 상태 === 3개이면 타입까지 비교해줌

--변수 var의 특징-- 1. 변수 이름을 중복해서 사용 가능 2. 변수 선언 전에 사용 가능(호이스팅) 3. 함수 스코프

--변수 let과 const의 특징-- 1. 변수 이름 중복선언 불가(문법오류) 2. 변수 선언 전에 사용 불가(참조오류) 3. 블록 스코프

JavaScript언어의 타입은 원시 값과 객체로 나눕니다. 원시값 : Boolean, Null, Undefined, Number, Bigint, String, Symbol 객체 : 속성의 컬렉션

데이터의 타입에는 문자형, 숫자형, 논리형 숫자형 ()
